# PRD — 직관생활

> 이 문서는 제품 요구사항·기능·방향을 담는 문서입니다. (코드 구조/컨벤션은 CLAUDE.md, DEV_GUIDE.md 참고)
> 최종 업데이트: 글래스 UI 전면 리디자인 + 포토카드 시스템 교체 + 인증 세션 해결 반영 시점.

## 1. 제품 비전
**한줄 요약**: 야구 직관을 기록하고, 예쁜 카드로 만들어 인스타에 바로 공유하는 앱
**타겟 유저**: 20~30대 여성 KBO 팬
**핵심 차별점**: 단순 기록이 아니라 "기록 → 예쁜 카드 → 공유"의 경험. (라이브 관전은 빅이닝 같은 앱의 영역이고, 우리는 기록·공유에 집중)

## 2. 디자인 시스템 (현재 확정 상태)
- **배경**: 퓨어 화이트 `#ffffff`
- **카드 표면**: 쿨 그레이 `#f1f3f6` (글래스 카드 `.glass-card`, 반투명 + blur + 그림자, 흰색 보더 하이라이트는 제거)
- **포인트(accent)**: 블루 (모든 화면 공통 토큰)
- **텍스트**: `text-text-primary/secondary/tertiary` 토큰
- **하단 탭바**: 공통 컴포넌트 `BottomNav variant="glass"` (메인 5탭 + 하위 페이지에도 적용)
- **폰트**: Pretendard
- 색·글래스는 모두 **globals.css 전역 토큰**으로 관리 (한 곳 수정 시 전 화면 반영)
- 글래스의 유리 질감은 약함(단색 배경 특성) → 현재 "클린 미니멀" 방향으로 수용
- 옛 스타일(`bg-[#EBF2FD]`, `#1A56DB`, 파스텔 카드, `text-slate-*`)은 제거 완료. **단 온보딩 화면은 아직 옛 톤이 남아 정리 예정.**

## 3. 완료된 기능 ✅

### 3-1. 이메일 로그인
- 이메일 OTP 코드 방식 (8자리 코드, 매직링크 아님 — 같은 페이지에서 코드 입력)
- Gmail SMTP 연동
- **세션 유지(iOS PWA 재로그인 문제 해결 완료)**: 세션을 localStorage에 저장(`persistSession`/`autoRefreshToken`), 부팅 가드는 `getSession()` 사용(콜드 스타트 튕김 방지), 랜딩(`/`)에 "세션 있으면 `/home`으로" 분기. 한 번 로그인하면 PWA를 닫았다 열어도 유지됨.
- 카카오 로그인: 사업자등록 → 비즈앱 전환 후 추가 예정 (KOE205로 현재 불가)

### 3-2. 온보딩 (최초 1회)
- 10개 구단 2열 그리드, 팀 선택 → Supabase `users` 테이블 저장, 로그인 가드
- ⚠️ UI 톤: 아직 옛 파스텔/파란 톤 (글래스 리디자인 미적용 — 백로그 항목)

### 3-3. 홈 화면 (글래스 톤)
- 인사말 + 팀 캐릭터(고양이)
- 오늘의/다가오는 경기 카드 (KBO 일정 연동, 종료 시 스코어 / 예정 시 일정) — **경기 시작 시간 표시 작업 진행 중**
- 시즌 요약 (직관 횟수, 승률 도넛 — DB 실시간)
- 최근 기록 카드
- "오늘의 직관 기록하기" 고정 CTA + 하단 글래스 탭바
- KBO 로딩 시 스켈레톤으로 레이아웃 시프트 방지

### 3-4. 직관 기록 작성 / 관리 (글래스 톤)
- 날짜 선택(기본 오늘), 구장 선택(9개), 날짜+구장 시 KBO 자동 매칭(상대팀·스코어 자동 채움)
- 홈/원정, 한줄 코멘트, 사진 업로드(최대 5장, Supabase Storage)
- 저장 → DB INSERT → **기록 상세로 이동** (이후 상세에서 "카드 만들기")
- 기록 목록 / 상세 / 수정 / 삭제
- 목록 디테일: 사진 썸네일 우측 고정 정사각 + 여러 장이면 장수 배지(대표 1장만), 승/패/무 배지 우측 상단 고정
- 작성·수정·상세 화면에 하단 탭바 추가

### 3-5. 인스타 카드 생성 ⭐ 킬러 기능 (새 포토카드 시스템)
- **StoryCard** 컴포넌트: 9:16 인스타 스토리 비율, 두 가지 모드
  - **채우기(fill)**: 사진 전체 배경 + 어두운 오버레이 위 정보 카드
  - **여백(matte)**: 사진 + 색 여백 + 아래 정보
- **CardEditor** 컨트롤: 모드 토글, 정보 위치(상/중/하), 오버레이 투명도(은은/기본/선명), 배경색 프리셋, 데코 토글, 기기 사진 업로드/선택(기록 사진 썸네일에서 고르기), 사진 비율로 모드 자동 추천
- EXIF 회전 보정(픽셀에 구워넣기), PNG 다운로드(**html2canvas-pro**), Web Share(모바일)
- 카드 만들기 화면 셸은 글래스 톤. ⚠️ 가운데 카드 미리보기(StoryCard)는 추출용 인라인 스타일이라 손대지 않음
- (구 파스텔 카드 `pastel-card.tsx`는 legacy, 미사용 — 추후 삭제)

### 3-6. 승률 통계 (글래스 톤)
- 전체 승률(도넛), 전적(N경기 N승 N패 N무), 홈/원정 승률 비교
- 최근 5경기 결과 dot (승=accent / 패·무=회색 계열, 결과 색 체계 통일)
- 상대팀별 전적 바 차트
- 상대팀 시점 통계 "혹시 나는 상대팀 승리요정?" — **승리요정/패배요정 둘 다 구현** (2경기 이상 비교 대상, 승리=패배요정이 같은 팀이면 패배요정 문구 생략)
- DB 실시간 계산

### 3-7. 달력 뷰 (글래스 톤)
- 월간 캘린더, 직관 날짜 결과별 dot(색 체계 통일)
- KBO 월간 일정 오버레이(상대팀 캐릭터)
- 홈/원정 구분(테두리 색)과 오늘 날짜 강조(숫자 칩)를 분리 표시
- 날짜 탭 → 기록 요약
- ⚠️ 백로그: 홈/원정 테두리 동그라미 제거 → 직관 경기 승/패/무를 캐릭터 뒤 배경색 큰 원으로 변경 예정

### 3-8. 마이페이지 (글래스 톤)
- 프로필 표시, 응원팀 변경(모달), 로그아웃

### 3-9. 팀 캐릭터
- 자체 고양이 캐릭터(AI 생성, 10팀 × 팀 컬러, 모자/나비넥타이로 구분), 배경 제거 처리 완료
- 구단 마스코트와 겹치지 않는 디자인

### 3-10. PWA
- manifest, 앱 아이콘, 홈 화면 추가 가능

## 4. 미구현 기능 / 백로그 (우선순위순)

### 진행 중 / 다음
0. **온보딩 화면 글래스 톤 정리** (남은 옛 스타일 마무리)
1. **경기 시작 시간 표시 마무리** — 홈(진행 중) + 달력
2. **종료 경기 이닝별 라인스코어(스코어보드)** — record 엔드포인트, 경기당 1회 수집, 기록 상세/카드에 표시
3. **라이브 스냅샷 범위 결정** — 가벼운 쪽(스코어+이닝) 우선. 주자 다이아몬드 풀스펙은 넣어도 진행 중 경기 상세에 한정
4. **통계 확장** — 다른 기록 앱들의 재미있는 통계 추가

### 고도화
5. **달력 표시 개선** — 홈/원정 동그라미 제거 → 승/패/무 캐릭터 배경 큰 원
6. **색상 / 응원팀 색 테마 + 전체 색 조화** — 마이 탭에서 팀 색 선택, accent를 팀 색으로
7. **화면 전환 속도 개선** — 딜레이/끊김 원인 진단(프리페치·페칭·캐싱)
8. **캐릭터 디자인 + 배치** — 단일 베이스 + 팀별 동물 모티프 액세서리(별도 트랙, 일러스트)

### 추후
- 카카오 로그인(비즈앱 전환 후), 집관 기록, 닉네임 설정, 오프라인 캐싱(서비스 워커)
- 구장 먹거리 가이드, 소셜 기능, 숏폼 영상, 유꾸마켓 연동

## 6. 데이터 소스 (KBO)
- 네이버 스포츠 비공식 API(`api-gw.sports.naver.com`), `kbo_games` 테이블 캐시
- `/schedule/games`(일정/결과, 시작 시각·상태 등 / 사용 중), `/record`(라인스코어·팀성적), `/preview`(상대전적·폼·순위), `/relay`(라이브 상황 — currentGameState)
- 비공식 API → 방어적 파싱 필요. record/preview/relay는 경기당 1콜 → 캐싱 필수

## 7. 라이선스 / 법적 주의사항 (절대 준수)
- KBO 구단 로고/마스코트/엠블럼 사용 **절대 금지**
- 구단 이름은 정보 전달 목적으로만 사용
- **선수 이름 사용 금지**(퍼블리시티권). 데이터도 선수 개인정보(라인업·박스스코어 등) 금지 — 경기·팀 단위 데이터만
- 유사 마스코트 제작 금지(부정경쟁방지법)
- 자체 고양이 캐릭터 + 팀 색상만 활용
