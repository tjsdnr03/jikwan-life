## 1. 프로젝트 개요

- **사업자**: SUNWOOK (김선욱, 삼성전자 System LSI 재직, 직장 병행)
- **서비스명**: 직관생활

- **사업 구조**: SUNWOOK이 앱 개발, 와이프가 유꾸마켓(네이버 스마트스토어) 운영
- **배포**: https://jikwan-life.vercel.app

### 관련 사업 계획
- 3대 아이템: ① 직관생활 앱 ② 원정 직관 큐레이션 ③ 유꾸마켓 (유니폼 꾸미기 커머스)
- 유꾸마켓은 와이프 전담, 자석뱃지/리본/키링 등 판매 예정

## 2. 기술 스택 & 환경

| 항목 | 내용 |
|------|------|
| Framework | Next.js 16 (App Router, Turbopack) |
| Styling | Tailwind CSS v4 |
| Backend/DB | Supabase (PostgreSQL + Auth + Storage) |
| Auth | 이메일 OTP 코드 (8자리, signInWithOtp + verifyOtp) |
| SMTP | Gmail SMTP (tjsdnr03@gmail.com, smtp.gmail.com:465) |
| Deployment | Vercel (GitHub 연동 자동 배포) |
| Language | TypeScript |
| Image Gen | html2canvas-pro (html2canvas 아님! Tailwind v4 oklch 호환) |
| Icons | lucide-react |
| KBO Data | 네이버 스포츠 API (api-gw.sports.naver.com) |
| 개발 도구 | Cursor Pro (Composer 2.5) + Claude Code (Opus) |

### 개발 환경 시작 방법
1. Cursor로 `jikwan-life` 폴더 열기
2. 터미널 탭 1: `npm run dev` (로컬 서버, localhost:3000)
3. 터미널 탭 2: `claude` (Claude Code)
4. 저장/배포: `git add -A && git commit -m "메시지" && git push` → Vercel 자동 재배포

### 도구 분업 원칙
- **Cursor Composer**: UI 컴포넌트, 페이지, 디자인 수정 (가성비 좋음, 한도 넉넉)
- **Claude Code**: 백엔드, API, 복잡한 로직, 인프라, 디버깅 (정확도 높음, 한도 제한)

- **Project URL**: https://mdwedauwltxphhdbesee.supabase.co

## 6. 중요한 기술적 결정/이슈

### html2canvas → html2canvas-pro
- Tailwind CSS v4가 oklch() 색상을 사용하는데, html2canvas 1.4.1은 이를 파싱 못 함
- html2canvas-pro로 교체하여 해결 (드롭인 포크, API 동일)
- **절대 html2canvas로 다시 바꾸면 안 됨**

### 카카오 로그인 KOE205 에러
- Supabase가 기본으로 account_email scope를 요청하지만, 개인 개발자 계정에서는 이메일 동의항목 설정 불가
- 비즈앱 전환(사업자등록 필요) 후 추가 예정
- 현재는 이메일 OTP 코드 방식으로 대체

### 로그인 방식 변경 이력
- 초기 계획: 카카오 OAuth → KOE205 에러로 불가
- 2차 시도: 구글 OAuth → Google Cloud Console 설정 필요하여 보류
- 3차 시도: 이메일 매직링크 → 작동하지만 새 창 열리는 UX 불편
- 최종: **이메일 OTP 코드 (8자리)** → 같은 페이지에서 코드 입력, UX 최적

### KBO 데이터
- 네이버 스포츠 API (api-gw.sports.naver.com/schedule/games) 사용
- 월간 범위 조회 지원 (fromDate~toDate 한 번 호출)
- kbo_games 테이블에 캐싱 (service_role 키 필요)
- 팀명 매핑: 네이버 팀명(삼성, LG 등) → 앱 팀 코드(lions, twins 등)

### KBO 라이선스 주의사항
- 구단 로고/마스코트/엠블럼 사용 절대 금지
- 구단 이름은 정보 전달 목적으로만 사용
- 선수 이름은 MVP에서 사용하지 않음 (퍼블리시티권 이슈)
- 유사 마스코트 캐릭터 금지 (부정경쟁방지법)
- 자체 고양이 캐릭터 + 팀 색상만 활용
- 유꾸마켓에서는 구단명을 상품명에 직접 사용 금지, "블루 시리즈" 등 색상으로만 구분

## 9. 사업 관련 메모

- 네이버 스마트스토어로 진행

### 캐릭터 전략
- 자체 고양이 캐릭터 (KBO 어떤 구단과도 겹치지 않음)
- 각 팀 색상 모자 + 나비넥타이로 구분
- 유사 구단 마스코트 만들지 않기 (법적 리스크)
- AI(Gemini)로 생성, 장기적으로 전문 일러스트레이터 의뢰 권장

### 카드 디자인 방향 (와이프 피드백)
- 파스텔톤 선호 (팀 컬러의 파스텔 버전)
- 3개 템플릿 계획: 파스텔(완성), 클래식(미구현), 볼드(미구현)
- 가로/세로 사진 모두 대응 필요
- 파스텔 템플릿이 가로/세로 모두 안전한 구조
